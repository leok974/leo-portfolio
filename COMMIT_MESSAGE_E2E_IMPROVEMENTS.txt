feat(e2e): improve CSV testing with deterministic href validation

## Summary

Enhanced E2E testing infrastructure with test-friendly CSV export and
optional live-mode testing capabilities.

## Changes

### 1. CSV Export: Button → Test-Friendly Anchor
- **File**: `src/components/OpsAgents.tsx`
- Changed CSV button to anchor tag with `data-testid="csv-link"`
- Deterministic `href` attribute constructed from filter state
- Same UX for users (target="_blank" opens in new tab)
- More reliable for E2E testing (no popup handling needed)

### 2. CSV Test: Popup → Href Validation
- **File**: `tests/e2e/ops-agents.spec.ts`
- Updated "download CSV link preserves filters" test
- Validates href attribute contains all filter parameters
- Assertions: status, task, limit parameters present in URL
- No popup window handling = more reliable and deterministic
- Test applies filters via UI, then checks CSV link href

### 3. CI: Optional Live-Mode Job
- **File**: `.github/workflows/e2e.yml`
- Added `e2e-live` job (runs on workflow_dispatch only)
- Starts real backend API and dev UI
- Runs tests with `E2E_LIVE=1` (no mocks)
- Uploads separate artifacts: playwright-report-live, playwright-traces-live
- Includes log output on failure for debugging
- Useful for pre-deployment smoke tests

### 4. Documentation Updates
- **Files**: `tests/e2e/README.md`, `E2E_TESTS_LOCKED.md`
- Updated test coverage table (CSV test description)
- Changed selector documentation: csv-btn → csv-link
- Added PowerShell commands for E2E_LIVE mode
- Documented e2e-live CI job usage
- Added CSV test improvement explanation

## Usage

### Run tests normally (mocked APIs)
```bash
npx playwright test tests/e2e/ops-agents.spec.ts
```

### Run tests in live mode (real backend)
```bash
# Windows PowerShell
$env:E2E_LIVE="1"; npx playwright test tests/e2e/ops-agents.spec.ts

# Linux/Mac
E2E_LIVE=1 npx playwright test tests/e2e/ops-agents.spec.ts
```

### Trigger live-mode in CI
1. Go to GitHub Actions → E2E workflow
2. Click "Run workflow" (manual dispatch)
3. The `e2e-live` job will run against real backend
